# Бронирование на сайте regarega.co — дизайн (заявка с подтверждением)

Дата: 2026-06-12
Поток D / демэнд-рельс, шаг 2-3. Связано: `project_rega_site_availability_prices`, `project_rega_os_v2_calendar_prices`, `project_rega_date_inquiry_lead`.

## Зачем

Превратить страницу квартиры на regarega.co из «витрины с кнопкой телеграма» в настоящий канал прямых броней. Гость видит живые даты и цену → оставляет заявку → оператор подтверждает в Rega OS. Это игла «% броней через нас» (мимо комиссии Airbnb): чем больше прямых броней, тем выше доля и ADR (стратегия `reference_agentic_strategy`).

## Модель процесса (решения залочены)

- **Тип:** заявка с подтверждением оператором («request to book»), НЕ мгновенная бронь. Каркас под будущую платёжку: «подтвердить» позже заменится «оплатить».
- **Где обрабатывается:** в Rega OS v2, заявка = бронь со статусом «заявка» (особый). Не отдельная система.
- **Контакт гостя:** имя + ОДИН канал на выбор (telegram / whatsapp / email). Минимум трения. Телефон/почту добираем при подтверждении.
- **Даты до подтверждения:** НЕ блокируются. Несколько гостей могут запросить одни даты; оператор подтверждает одну, остальным пишет «занято».

## Гостевая часть — панель бронирования (макеты одобрены)

Встроена в существующую sticky-панель `.apt-book` справа на странице квартиры. Стиль = реальный бренд сайта: cream-фон, тонкие линии, сланцевый акцент `--page:#3A4868`, шрифты Bricolage Grotesque (заголовки строчными) + JetBrains Mono (метки uppercase tracked), один сильный элемент — тёмная pill-кнопка. Воздух, минимум заливки.

**Состояние 1 — даты не выбраны:**
- Цена «from $X / ночь» (минимальная из доступных) + подпись «≈10% дешевле, чем на airbnb».
- Поля «заезд — выезд» (пусто, «добавьте даты») и «гости».
- Мягкая пунктирная подсказка-кнопка «выберите даты ↑».
- 3 шага: «выберите даты → увидите цену», «оставьте заявку — имя и контакт», «ответим за час — живой человек».
- Внизу одна тихая ссылка: «хотите сперва спросить? ↗ telegram».

**Состояние 2 — даты выбраны (форма заявки раскрывается inline, компоновка A):**
- Точная цена за ночь.
- Поля дат (заполнены) и гостей.
- Итог: «N ночей — $XXX».
- Блок «— куда вам ответить»: имя + сегмент-переключатель telegram/whatsapp/email + поле контакта.
- Тёмная кнопка «↗ отправить заявку», под ней «ответим в течение часа — живой человек».
- Та же одна telegram-ссылка.

**Календарь выбора дат (без светофора — монохром, одобрено):**
- НЕ красно-зелёный. Цвет почти не используется, в тон бренду сайта.
- **Свободная ночь:** число + цена за ночь под ним (из базы ×1.1), обычный текст.
- **Занятая ночь:** число аккуратно ЗАЧЁРКНУТО тонкой линией (line-through, thickness ~1.5px, цвет --line-strong), приглушено серым (--faint), без цены, некликабельно (cursor:not-allowed). Гость физически не может выбрать занятое → заявка на занятое невозможна в принципе.
- **Выбранный диапазон:** заливка СЛАНЦЕВЫМ акцентом сайта `--page:#3A4868` (НЕ чёрный); края диапазона — насыщенный сланец, серединка — светлый сланец (--page-soft ~#E4E8EF). Это единственный сильный элемент.
- **Сегодня:** маленькая оранжевая точка `--orange:#FF5E1A` под числом (фирменная деталь).
- **Прошлое:** бледное, некликабельно.
- **Наведение** на свободный день — тонкая сланцевая рамка.
- Лаконичная подпись вместо легенды-светофора: «— свободно · цена за ночь» и «0̶0̶ занято».
- «Занято» = подтверждённые брони + блоки оператора. Чужие неподтверждённые заявки НЕ помечаются занятыми (даты честно свободны до подтверждения).
- Минимум 2 ночи (как сейчас в house rules).

**Экран после отправки:** «заявка отправлена! оператор ответит в течение часа — живой человек, не бот. даты пока свободны для других — кто подтвердится, тот и бронирует.»

## Цены — модель

- В базе лежат НЕТТО-цены оператора (nightly_rates переопределения → base_rate из apartment_economics). Сайт показывает **×1.1** (+10%, наценка платформы под будущий эквайринг). Наценка = константа на стороне сайта; база всегда нетто.
- Нужна публичная отдача цен анониму (вью/RPC, только цены — НЕ суммы броней), по образцу `public_availability` (миграция 005). Новая миграция в `~/rega-os-v2/migrations/`.

## Серверная часть заявки

- Гость жмёт «отправить заявку» → POST на serverless-эндпоинт (по образцу `~/Downloads/rega-batumi/api/contact.js`, бот `@regarega_contact_bot` уже на проде).
- Эндпоинт: (а) пишет в базу новую бронь со статусом «заявка» (operator_id квартиры, apartment_id, ci/co, гости, имя, контакт-канал+значение, цена-снимок); (б) шлёт дубль-уведомление в Telegram оператору.
- Honeypot + лимит частоты (как в contact.js). Без капчи.
- ⚠️ Запись брони анонимом — НЕ напрямую из браузера (RLS закрыт). Через serverless с сервисным доступом ИЛИ SECURITY DEFINER RPC `create_booking_request(...)`, который создаёт ТОЛЬКО заявку (статус жёстко «заявка», amount в выручку не идёт). Решить на этапе плана; безопаснее RPC с белым списком полей.

## Операторская часть (Rega OS v2)

- Заявка = бронь, новый статус «заявка» (отдельно от fin_status; не путать с source='block'). Нужна колонка/значение статуса заявки + флаг, что это с сайта.
- **Дашборд «сегодня» → «требует внимания»:** строка «новая заявка с сайта — {квартира} · {даты} · {гости} · {имя}», подсвечена.
- **Календарь:** ночи заявки — пунктиром (запрошено, не подтверждено), НЕ блокируют реальные брони, не красные на сайте.
- **Telegram-дубль** оператору.
- Открыть заявку → даты/гости/имя/контакт/цена + кнопки **«подтвердить»** (→ обычная бронь, даты закрываются = зачёркиваются на сайте, гость заносится в guests) / **«отклонить»** (заявка удаляется/архивируется).
- Заявка НЕ считается в P&L/выручку до подтверждения.
- Связь с гостем — по оставленному контакту (вручную).

## Крайние случаи

- Две заявки на одни даты — обе висят, подтверждается одна, второй «занято».
- Заявка на занятое — невозможна (занятое зачёркнуто и некликабельно на сайте).
- Повисшая заявка — остаётся в списке; показывать «N дней без ответа».
- Спам/боты — honeypot + лимит, без капчи.

## Границы (НЕ в этой задаче)

- Оплата на сайте (платёжка/эквайринг) — будущее; дизайн оставляет под неё каркас (статус «заявка» → «оплачено»).
- Импорт броней/цен с Airbnb — отдельные задачи (`project_rega_os_v2_calendar_prices`).
- Отдача на bre.ge (тот же механизм, отдельным проходом).
- RU-копия страницы (`aptDetailRu.ts`) — править симметрично EN.
